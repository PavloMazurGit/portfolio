Weapon System

Weapon Hit

Weapon Configuration

The weapon system handles all player attack logic, including hit detection, damage calculation, and weapon behavior configuration.

The Weapon Hit function is responsible for detecting collisions between weapons and entities, applying damage, and triggering effects such as knockback or hit animations.

The Weapon Configuration defines each weapon’s properties such as attack speed, damage values, range, and special effects, allowing easy balancing and customization of different weapon types.

Together, these systems ensure responsive combat and flexible weapon design that can be expanded with new mechanics in the future.

Render

Render Resolution

Render Update Loop

The render system is responsible for drawing all game elements to the screen, including players, objects, and the environment.

It runs separately from the game logic to ensure smooth and consistent frame updates, typically based on the current frame rate.

The render resolution system controls how the game is displayed on different screen sizes, ensuring visuals stay sharp and properly scaled.

The render update loop continuously redraws the scene based on updated game states, keeping visuals synchronized with gameplay changes.

Together, these systems ensure smooth performance and visually consistent gameplay across all devices.

Anticheat

Anticheat Obfuscator Input

Anticheat Obfuscator Output

The anticheat system is designed to protect the game from unauthorized modifications and suspicious behavior by analyzing and securing core scripts.

The obfuscator input represents the original source code before protection is applied. This version is readable and structured for development purposes.

The obfuscator output shows the processed result after obfuscation, where the code is transformed into a more complex and harder-to-understand format to prevent reverse engineering.

This system helps improve security by making it significantly more difficult for exploiters to modify or analyze the game’s internal logic.

GUI Element

Gui List Files

The GUI system handles all in-game interface elements, including menus, panels, and interactive components. It is designed to be modular and easy to manage.

The GUI Controller manages overall interface logic, including opening, closing, and switching between different UI screens.

The GUI Layout defines how interface elements are structured and positioned on the screen to ensure a consistent user experience.

The GUI Elements represent individual UI components such as buttons, panels, and text fields, each responsible for specific interactions.

The GUI List Files system organizes and stores UI-related files, making the interface scalable and easier to maintain across the project.

Together, these systems form a flexible and modular GUI architecture that supports clean design and efficient UI management.
